접속
  1) conn
  2) conn 계정
  3) conn 계정/비밀번호
-----------------------------------------------------------------------
접속자 확인
  show user
-----------------------------------------------------------------------
사용자 생성
  1) 관리자권한(DBA)을 가진 계정에서만 할 수 있음
  2) 같은 계정은 한 번만 생성할 수 있다.
  3) create user 계정이름 identified by 비밀번호;
-----------------------------------------------------------------------
사용자에게 권한 부여
  1) 권한이 없으면 아무 것도 하지 못한다.
  2) connect, resource, dba 중 원하는 권한을 부여한다.
  3) 관리자권한(DBA)을 가진 계정에서만 할 수 있음
  4) grant 권한 to 계정;
-----------------------------------------------------------------------
사용자 삭제
  1) 관리자권한(DBA)을 가진 계정에서만 할 수 있음
  2) 데이터가 없는 사용자 삭제
    2-1) drop user 계정;
  3) 데이터가 있는 사용자 삭제
    3-1) drop user 계정 cascade;
-----------------------------------------------------------------------
사용자의 권한 해제
  1) 관리자권한(DBA)을 가진 계정에서만 할 수 있음
  2) revoke 권한 from 계정;
-----------------------------------------------------------------------
Oracle의 데이터 타입
  1) 숫자: number
  2) 고정형 문자: char  (우편번호, 주민등록번호 등)
  3) 가변형 문자: varchar2  (주소 등)
  4) 날짜: date, timestamp
-----------------------------------------------------------------------
Oracle의 타입과 Java의 타입 매칭
       oracle           java
  1)  number     int, double
  2)    char           String
  3)  varchar2        String
  4)    date      java.sql.Date
  5) timestamp java.sql.Timestamp
-----------------------------------------------------------------------
Oracle의 테이블은 Java의 클래스와 매칭된다.
  1) Java Class
     class Member {
         String id;
         int point;
         Date regDate;
     }

  2) Oracle Table
     create table member (id varchar2(20), point number, regDate date);
-----------------------------------------------------------------------
키(key)
  1) 기본키: primary key
    - 중복이 없고, (unique)
    - 널 값이 없다. (not null)
    - 개체 무결성이라고 한다.
    - 예시: 대기번호, 주민번호, 자동차번호, 학번, 군번 등
  2) 외래키: foreign key
    - 다른 테이블의 기본키 또는 unique 칼럼을 참조하는 키(key)이다.
    - 참조(reference)하는 키나 칼럼이 가지고 있는 값(도메인)만 가질 수 있다.
    - 참조 무결성이라고 한다.
-----------------------------------------------------------------------
Table 생성
  1) 기본키, 널 값 유무(필수 속성), 중복 가능 유무(unique)를 판단한다.
  2) 다른 테이블을 참조할 때는기본키 또는 unique 칼럼을 참조한다.
  3) create table 테이블명(칼럼명 타입 [not null] [unique] [primary key])
     create table 테이블명(칼럼명 타입 references 참조체이블명(참조칼럼))
-----------------------------------------------------------------------
Table 삭제
  1) 참조 관계의 테이블들을 삭제하는 경우 반드시 '외래키'를 가진 테이블을 먼저 지워야 한다.
  2) drop table 테이블명;
